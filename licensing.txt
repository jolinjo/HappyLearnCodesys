f:T14db,# CODESYS RTE 環境與支援硬體

> 資料來源：CODESYS 官網與說明文件，供開發選型參考。

---

## 一、RTE（Runtime Environment）簡介

CODESYS Control RTE（Real-Time Environment）是執行於工業 PC 或嵌入式控制器上的**軟體 PLC 即時執行環境**，可搭配 CODESYS Development System 以 IEC 61131-3 進行程式開發與下載。

- **Control RTE SL**：適用於 **Windows** 的即時軟體 PLC（如 Beckhoff CX 等）。
- **Control for Linux ARM SL**：適用於 **Linux（Debian 系）** 的 ARM 平台執行環境。
- **Control Runtime Toolkit**：供裝置製造商將 CODESYS Control 移植到**自訂硬體**的開發套件。

---

## 二、開發環境中的設備類型（RTE / Win 與 32/64 位元）

在 CODESYS 專案中新增設備時，於 **PLC → SoftMotion PLC** 下會看到四種目標，差異如下。

### 2.1 RTE 與 Win 的差異

| 設備名稱 | 說明 | 適用情境 |
|----------|------|----------|
| **CODESYS Control RTE V3** | **RTE = Real-Time Extension**。使用即時擴充（如 IntervalZero RTX、Beckhoff 即時核心），PLC 掃描週期具**即時性、可預測**。 | 現場控制、運動控制、對掃描時間要求嚴格的應用；Beckhoff CX 等即時 Windows 控制器。 |
| **CODESYS Control Win V3** | **Win = 標準 Windows**。在一般 Windows 上執行，**無即時核心**，掃描週期會受 Windows 排程影響。 | 本機模擬、開發除錯、不需硬即時的應用、或沒有安裝 RTE 的 PC。 |

**簡單區分**：  
- 要**即時、穩定週期** → 選 **RTE**（且硬體需已安裝對應的 RTE 套件）。  
- 要**在一般 PC 上跑模擬/開發** → 選 **Win**。

### 2.2 32 位元（V3）與 64 位元（V3 x64）的差異

| 設備名稱 | 架構 | 說明 |
|----------|------|------|
| **CODESYS Control RTE V3** / **CODESYS Control Win V3** | **32 位元** | 程式與資料位址空間受 32 bit 限制；部分現場匯流排或驅動僅提供 32 bit 版。 |
| **CODESYS Control RTE V3 x64** / **CODESYS Control Win V3 x64** | **64 位元** | 可定址較大記憶體，適合大型專案；需在 64 位元 Windows 上執行，部分附加元件可能僅支援 32 bit。 |

選擇時需與**實際執行環境**一致：若目標機是 64 位元 Windows 且已安裝對應 RTE，應選 **RTE V3 x64**；若僅在本機用一般 Windows 模擬，可選 **Win V3** 或 **Win V3 x64**。

### 2.3 四種設備對照表

| 設備 | 即時性 | 位元 | 典型用途 |
|------|--------|------|----------|
| CODESYS Control RTE V3 | 即時（RTE） | 32 bit | 即時控制、32 bit 硬體或驅動 |
| CODESYS Control RTE V3 x64 | 即時（RTE） | 64 bit | 即時控制、64 bit 工業 PC |
| CODESYS Control Win V3 | 非即時（標準 Windows） | 32 bit | 本機模擬、開發 |
| CODESYS Control Win V3 x64 | 非即時（標準 Windows） | 64 bit | 本機模擬、開發、大記憶體需求 |

## 三、支援的作業系統與 CPU 架構

| 平台類型 | 作業系統 | CPU 架構 | 說明 |
|----------|----------|----------|------|
| **Windows** | Windows 8 / 10 / Embedded Windows | x86（單核）、x64（多核） | CODESYS Control RTE SL，適用 PC-based 控制器 |
| **Linux** | Debian-based Linux（32/64 bit） | ARMv7、ARMv8 | CODESYS Control for Linux ARM SL |
| **Linux** | 標準 Linux | x86、ARM | 透過 Runtime Toolkit 或 Store 現成 RTE 產品支援 |
| **自訂平台** | 依實作而定 | x86、ARM 等 | 使用 CODESYS Control Runtime Toolkit 移植 |

**注意**：RTE 不適合在**容器（Container）**或**虛擬機器（VM）**內執行，需在實體或嵌入式硬體上運行。

---

## 四、具體硬體與介面支援

### 4.1 CODESYS Control RTE SL（Windows / Beckhoff CX）

- **適用硬體**：Beckhoff CX 系列嵌入式 PC、一般 x86/x64 工業 PC（執行 Windows）。
- **乙太網路晶片**：Realtek、Intel 等常見晶片組。
- **CAN 介面卡**（選配）：
  - **Peak**：PCIe、MiniPCIe
  - **Ixxat**：SJA1000（PCI）
  - **Automata**：全系列 PCI（1/2 通道）

### 4.2 現場匯流排（Fieldbus）支援

RTE 可搭配對應授權與設備，支援下列通訊協定（依產品與授權而異）：

| 類型 | 協定 |
|------|------|
| **乙太網路型** | EtherCAT、PROFINET、EtherNet/IP、Sercos III（部分僅 32 bit） |
| **現場匯流排** | CANopen、Modbus TCP/Serial、PROFIBUS、J1939 |
| **其他** | CAN-PCI 卡（Peak、Ixxat、Automata）、OPC UA Server |

### 4.3 裝置製造商與自訂硬體

- 透過 **CODESYS Control Runtime Toolkit** 可將執行環境移植到**非標準平台**。
- 執行系統可依硬體能力做**模組化調整**，Store 中也有針對常見標準平台的**現成 SoftPLC** 可選購。

> 資料來源：CODESYS 官網與 [CODESYS Store](https://store.codesys.com/en/licenses.html)。  
> **匯率說明**：以下台幣（TWD）以 **1 EUR ≈ 35 TWD** 估算，僅供參考；實際請以購買時匯率與 Store 標價為準。

## 一、授權容器（License Container）

CODESYS 授權可儲存於下列兩種容器之一：

| 類型 | 說明 |
|------|------|
| **硬體 dongle（CODESYS Key）** | USB 硬體鎖，可儲存最多約 4,000 筆授權，並具加解密功能。需另購 CODESYS Key（約 €40）。 |
| **軟體授權容器** | 與 CODESYS 安裝時綁定於指定裝置（如開發 PC 或控制硬體），之後固定於該裝置。 |

啟用授權需網路連線；亦可透過 [License Central](https://license.codesys.com/) 或開發系統內分步完成啟用。

## 二、CODESYS 開發系統（兩種差異）

### 2.1 免費版：CODESYS Development System

- **費用**：**免費**，僅需同意使用者授權條款即可安裝。
- **內容**：IEC 61131-3 標準編輯與除錯、邏輯/運動/視覺化等基本工程功能，可開發並下載至支援的控制器。
- **適用**：個人學習、小型專案、不需團隊協作或進階工程工具時。

### 2.2 訂閱版：Professional Developer Edition (PDE)

- **性質**：**加購模組**，需先安裝 CODESYS Development System，PDE 為其上之**訂閱制**擴充套件。**PDE 僅提供訂閱制，無買斷（永久）授權**；訂閱到期未續費，PDE 功能即無法使用。
- **內容**：在開發系統基礎上增加專業團隊開發與程式品質工具，例如：
  - **原始碼版本控制**：整合 **Git**、**Apache Subversion (SVN)**（SVN 已規劃逐步停售，新專案建議用 Git）
  - **靜態程式碼分析**
  - **UML 編輯器**
  - **測試自動化工具**
  - **執行時間量測（Profiling）**
- **授權與價格**（訂閱制，依 Store 為準）：

| 授權類型 | 歐元（EUR/年） | 約台幣（TWD/年） | 說明 |
|----------|----------------|------------------|------|
| **PDE 單用戶** | 約 €550 | 約 19,250 | 1 台開發機，1 年訂閱 |
| **PDE 網路授權（5 用戶）** | 約 €2,750 | 約 96,250 | 5 人共用，1 年訂閱 |

- **PDE 單用戶授權儲存方式（可二選一）**：  
  CODESYS 授權（含 PDE）可存於兩種容器，**購買／啟用時可選擇**：
  - **Dongle（CODESYS Key）**：授權寫入 USB 硬體鎖。**可攜帶、換電腦**：插上不同 PC 即可在該機使用，無需重新綁定。若尚未擁有 CODESYS Key，需另購（約 €40）。
  - **軟體授權容器（Software License Container）**：授權在首次啟用時**綁定當時那台電腦**，之後僅能在該機使用，**無法轉移到其他電腦**；換機通常需聯絡 CODESYS 支援處理。  
  若希望「一台授權、多台電腦輪流用」，請選擇將 PDE 存於 **CODESYS Key**。

- **訂閱期間可否更新開發環境？**  
  - **CODESYS 開發環境（Development System）本體**：**免費**，可隨時從官網或 CODESYS Installer 下載並安裝新版本，與是否擁有 PDE 無關。  
  - **PDE 附加工具**：在**訂閱有效期間**，訂閱內容包含該期內之**功能更新與升級**；只要持續續訂，即可搭配新版 CODESYS 使用並取得 PDE 工具更新。  
  - **訂閱到期後**：PDE 功能（Git/SVN、靜態分析、UML、測試、Profiler 等）將無法使用；**開發環境本體**仍可照常更新（免費），但無法使用 PDE 擴充功能。  
  （因 PDE 無買斷制，不存在「買斷後能否更新」；若改為「訂閱 + dongle」：訂閱有效即可更新，訂閱中斷則僅能使用免費開發環境。）

- **適用**：需版本控制、程式審查、測試自動化或多人協作的團隊開發。

### 2.3 兩種開發系統對照

| 項目 | CODESYS Development System（免費） | Professional Developer Edition (PDE) |
|------|-------------------------------------|--------------------------------------|
| **費用** | 免費 | 訂閱制（單用戶約 €550/年，5 人約 €2,750/年） |
| **程式編輯 / 除錯 / 下載** | ✅ | ✅（同一套開發系統） |
| **版本控制（Git / SVN）** | ❌ 需另裝外部工具 | ✅ 整合 Git、SVN（SVN 逐步停售） |
| **靜態分析 / UML / 測試自動化 / Profiling** | ❌ | ✅ |
| **適合對象** | 學習、小型專案、單人開發 | 團隊開發、需版本管理與程式品質工具 |

## 三、版本控制模組（含於 PDE，個別取得方式）

版本控制功能主要透過 **PDE** 取得；若只想要「版本控制」而不買整套 PDE，可留意下列產品是否在 Store 單獨販售（依當時 Store 為準）。

| 產品 | 說明 | 價格／取得方式 |
|------|------|----------------|
| **CODESYS Git** | 在開發環境內整合 Git，對所有應用物件做版本控制，無需本機另裝 Git。 | **免費**提供，或隨 PDE 一併使用。若 Store 有單獨項目則以 Store 為準。 |
| **CODESYS SVN** | 整合 Apache Subversion，對所有應用物件做版本控制。 | 已進入**服務期、規劃停售**，僅建議既有專案使用；新專案建議改用 **CODESYS Git**。若可單獨購買，價格請見 [CODESYS Store](https://store.codesys.com/en/catalogsearch/result/?q=svn) 工程類附加模組。 |

**注意**：若需「版本控制 + 靜態分析 + UML + 測試自動化」等完整工具，直接訂閱 **PDE** 較划算；若僅需 Git，可優先使用免費的 **CODESYS Git**（若您使用的 CODESYS 版本有內建或提供下載）。

### 5.4 安全控制（Safe Control）

#### 什麼是 Safe Control？什麼情境需要？

**Safe Control** 是 **功能安全（Functional Safety）** 用的 PLC 授權，用來做 **IEC 61508 SIL2 / SIL3** 等級的安全控制，保護**人員與設備**。

- **用途**：在標準邏輯控制之外，多一層「安全程式」：例如緊急停機（E‑Stop）、安全限速（SLS）、安全限位（SLP）、安全煞車控制（SBC）、安全門、光柵、雙手按鈕等，透過**安全連線**與安全驅動器、安全閥、安全 I/O 等設備通訊。
- **「安全連線數」**：每一條與**安全相關設備／安全功能**的邏輯連線算一筆（例如一個安全驅動的一組安全命令、一個安全閥、一條安全匯流排通道等）。授權等級依可用的安全連線數量分級，連線數越多、可接的安全裝置／功能越多。
- **常見使用情境**：
  - **機械製造**：工具機、包裝機、產線的緊停、安全門、光柵、區域監控。
  - **移動機械**：工程車輛、農機、 AGV 的安全速度、安全轉向、安全停車。
  - **能源／製程**：安全關斷、安全閥、燃燒控制。
  - **軌道、船舶、樓宇**：安全聯鎖、逃生與監控。

若法規或客戶要求 **SIL2/SIL3、PLe、Category 3/4** 等認證，或需與 PROFIsafe、FSoE（EtherCAT 安全）等安全協定整合，即需使用 **CODESYS Safety** 與對應 **Safe Control** 授權。

- 各廠 **AXIS_REF_*** 擴充庫：對應 EtherCAT、PROFINET 等驅動，延伸 `AXIS_REF_SM3`。

## 三、檔案類（File）

### 3.1 SysFile（系統底層檔案）

提供 Runtime 底層檔案操作，同步、低階 API。

| 類型 | 名稱 | 說明 |
|------|------|------|
| **Function** | `SysFileOpen` | 開啟檔案 |
| **Function** | `SysFileClose` | 關閉檔案 |
| **Function** | `SysFileRead` | 讀取 |
| **Function** | `SysFileWrite` | 寫入 |
| **Function** | `SysFileSetPos` | 設定讀寫位置 |
| **Function** | `SysFileGetPos` | 取得目前位置 |
| **Function** | `SysFileGetSize` / `SysFileGetSizeByHandle` | 取得檔案大小 |
| **Function** | `SysFileCopy` | 複製檔案 |
| **Function** | `SysFileDelete` / `SysFileDeleteByHandle` | 刪除檔案 |
| **Function** | `SysFileRename` | 重新命名 |
| **Function** | `SysFileTruncate` | 截斷檔案 |
| **Function** | `SysFileFlush` | 寫入緩衝區 |
| **Function** | `SysFileEOF` | 是否到達檔尾 |
| **Function** | `SysFileGetName` / `SysFileGetName2` | 取得檔名 |
| **Function** | `SysFileGetPath` | 取得路徑 |
| **Function** | `SysFileGetStatus` / `SysFileGetStatus2` | 取得狀態 |
| **Function** | `SysFileGetTime` | 取得檔案時間 |

**列舉/結構**：`ACCESS_MODE`、`SYS_FILE_STATUS`、`SYS_FILETIME` 等。

- 文件：[SysFile](https://content.helpme-codesys.com/en/libs/SysFile/Current/index.html)

---

### 3.2 SysFileAsync（非同步檔案）

非同步檔案 I/O，避免阻塞 PLC 週期，介面與 SysFile 類似但為非同步呼叫。

- 文件：[SysFileAsync](https://content.helpme-codesys.com/en/libs/SysFileAsync/Current/index.html)

---

### 3.3 CAA File（CAA 標準檔案與目錄）

CAA 標準化檔案與目錄存取，支援本機路徑、FTP、HTTP 等。僅支援二進位模式。

#### 檔案（File）

| 類型 | 名稱 | 說明 |
|------|------|------|
| **FB** | `FILE.Open` | 開啟檔案 |
| **FB** | `FILE.Close` | 關閉檔案 |
| **FB** | `FILE.Read` | 讀取 |
| **FB** | `FILE.Write` | 寫入 |
| **FB** | `FILE.SetPos` | 設定位置 |
| **FB** | `FILE.GetPos` | 取得位置 |
| **FB** | `FILE.GetSize` | 取得大小 |
| **FB** | `FILE.GetTime` | 取得時間 |
| **FB** | `FILE.GetAttribute` | 取得屬性 |
| **FB** | `FILE.Copy` | 複製 |
| **FB** | `FILE.Delete` | 刪除 |
| **FB** | `FILE.Rename` | 重新命名 |
| **FB** | `FILE.Flush` | 寫入緩衝區 |
| **FB** | `FILE.EOF` | 是否檔尾 |

#### 目錄（Directory）

| 類型 | 名稱 | 說明 |
|------|------|------|
| **FB** | `DirOpen` | 開啟目錄 |
| **FB** | `DirClose` | 關閉目錄 |
| **FB** | `DirList` | 列出目錄內容 |
| **FB** | `DirCreate` | 建立目錄 |
| **FB** | `DirCopy` | 複製目錄 |
| **FB** | `DirRemove` | 刪除目錄 |
| **FB** | `DirRename` | 重新命名目錄 |

#### 函式

| 類型 | 名稱 | 說明 |
|------|------|------|
| **Function** | `FILE.GetProperty` | 查詢實作能力（如是否支援 FTP/HTTP） |

路徑可為本機絕對/相對、`file://`、`ftp://`、`http://` 等（依實作而定）。

- 文件：[CAA File](https://content.helpme-codesys.com/en/libs/CAA%20File/Current/index.html)

## 四、時間類（Date / Time）

### 4.1 SysTimeRtc（即時時鐘）

存取目標系統 RTC（日期/時間）。

| 類型 | 名稱 | 說明 |
|------|------|------|
| **Function** | `SysTimeRtcGet` | 取得目前日期時間 |
| **Function** | `SysTimeRtcSet` | 設定 RTC |
| **Function** | `SysTimeRtcConvertDateToUtc` / `SysTimeRtcConvertUtcToDate` | 日期與 UTC 轉換 |
| **Function** | `SysTimeRtcConvertLocalToUtc` / `SysTimeRtcConvertUtcToLocal` | 本地與 UTC 轉換 |
| **Function** | `SysTimeRtcGetTimezone` / `SysTimeRtcSetTimezone` | 時區取得/設定 |
| **Function** | `SysTimeRtcControl` | RTC 控制 |
| **Function** | `SysTimeRtcHighResGet` / `SysTimeRtcHighResSet` | 高解析度時鐘 |
| **Function** | `SysTimeRtcConvertDateToHighRes` / `SysTimeRtcConvertHighResToDate` | 與高解析度時間轉換 |

**結構**：`RTS_SYSTIMEDATE`（年/月/日/時/分/秒/毫秒/星期/一年中第幾天等）。

- 文件：[SysTimeRtc](https://content.helpme-codesys.com/en/libs/SysTimeRtc/3.5.17.0/index.html)

---

### 4.2 SysTimeCore（高解析度時間間隔）

取得高解析度時間戳或經過時間（毫秒/微秒/奈秒等級），用於量測與計時。

| 類型 | 名稱 | 說明 |
|------|------|------|
| **Function** | `SysTimeGetMs` | 毫秒解析度 |
| **Function** | `SysTimeGetUs` | 微秒解析度 |
| **Function** | `SysTimeGetNs` | 奈秒解析度（依平台） |

（實際名稱以各版 SysTimeCore 文件為準。）
