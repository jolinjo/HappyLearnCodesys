# Security Best Practices 安全性實務建議 教學整理

> 本文件基於 CODESYS Offline Help 中下列主題整理而成：
> - `Security`（`_cds_struct_security.html`）
> - `Dialog: Project Settings: Security`（`_cds_dlg_project_settings_security.html`）
> - `Encrypting Communication, Changing Security Settings`（`_cds_encrypting_communication_changing_security_policy.html`）
> - `Client Does Not Support Security Feature`（`_cds_sec_faq_clients_not_supporting_security.html`）
> 目標是從實務角度整理開發系統、Runtime/PLC 與 WebVisu 的安全性要點與常見 FAQ，而非逐字翻譯。

---

## 1. 為什麼要重視安全性

控制器與產線聯網程度提高，潛在威脅也隨之增加。**安全性措施**用來保護資料與通訊管道，避免未授權存取與竄改，應納入專案與維運規劃。

CODESYS 與控制器提供多層安全功能，包含：專案保護（密碼／Dongle／憑證）、與控制器的加密通訊、裝置端使用者管理。以下依「專案保護」「通訊加密與使用者管理」「常見問題」整理實務建議。

---

## 2. 專案保護（Project Settings: Security）

**入口**：**Project → Project Settings** → **Security** 分類。

### 2.1 保護等級

| 選項 | 說明與建議 |
|------|------------|
| **No protection** | 專案檔未加密、未做完整性檢查；**強烈建議**改為至少啟用 Integrity check 或 Encryption。 |
| **Integrity check** | 新專案預設啟用。專案以專有格式儲存，載入時檢查完整性；**未加密**，若需保護內容請再啟用加密。 |
| **Encryption** | 啟用後可選擇 **Password**、**Dongle** 或 **Certificates** 加密專案內容。 |

**重要**：若使用**密碼加密**，遺失密碼將**無法再開啟或還原**專案，請妥善保管並考慮備援方式（例如 Dongle 或憑證）。

### 2.2 加密方式

- **Password**：設定並確認加密密碼；儲存後每次開啟專案（含作為 Library 參考載入）皆需輸入密碼。
- **Dongle**：需連接 CODESYS 安全鎖（dongle）到電腦。在 **Add** 中選擇已連接的 dongle 並加入 **Registered dongles**。載入專案時（含 Library 參考）也需插上該 dongle。
- **Certificates**：以憑證加密專案；**所有會共用此專案的使用者**，其憑證皆須已安裝在本機。可透過 **Certificate Selection** 對話框選擇憑證。

---

## 3. 與控制器的加密通訊與使用者管理

### 3.1 資料保護建議（組織與技術）

- 盡量**避免**將 PLC 與控制網路直接暴露於公網或 Internet；遠端存取建議透過 **VPN** 等額外資料鏈結層保護。
- 使用 **防火牆**、限制僅授權人員存取、採用**高強度密碼**；**出廠預設密碼**應在試車時變更，並定期更換。
- 善用 CODESYS 與控制器支援的安全功能：**與控制器的通訊加密**、**裝置端使用者管理**（限制登入與權限）。

### 3.2 通訊原則（Communication Policy）

在裝置編輯器的 **Communication Settings** 分頁，或 **Device → Change Communication Policy** 可設定：

- **加密**：Optional encryption / **Enforced encryption** / No encryption。
- **使用者管理**：Optional user management / **Enforced user management**。

啟用 **Enforced encryption** 或 **Enforced user management** 後，未符合條件的連線（例如未安裝信任憑證、未登入裝置使用者）將無法建立。

### 3.3 首次連線：安裝信任憑證與建立管理員

當控制器已強制加密與使用者管理，但尚未設定個別密碼與憑證時，典型流程為：

1. 在 Device Tree 雙擊控制器 → 開啟裝置編輯器 → **Communication Settings** → **Scan Network**，選擇控制器。
2. 若出現「控制器憑證未受信任」提示，可選擇將該憑證安裝到本機 **Controller Certificates** 儲存區，或僅接受本次連線。安裝後 30 天內有效，便於後續改為長期憑證。
3. 若提示啟用使用者管理，選擇 **Yes** 後在 **Add Device User** 中建立**第一個裝置管理員**（名稱、密碼；僅 **Administrator** 群組可選）。
4. 在 **Device User Logon** 中輸入剛建立的帳密完成登入；之後可在 **Users and Groups** 分頁管理裝置使用者。

本機已安裝的控制器憑證可在 Windows **certmgr.msc** → **Controller Certificates** 中檢視。

### 3.4 長期憑證與加密設定

- **建議**：安裝 **CODESYS Security Agent** 後，在 Security Screen 的 **Devices** 分頁設定控制器憑證，可產生較長效期（例如 365 天）的自簽憑證，或匯入 CA 簽署憑證。
- **無 Security Agent 時**：可在裝置編輯器 **PLC Shell** 分頁以指令操作（例如 `cert-getapplist`、`cert-genselfsigned`），步驟較多，詳見 Offline Help「Encrypting Communication」。
- 在 **Security Screen → User** 分頁可勾選 **Enforce encrypted communication**，強制所有控制器連線加密；或僅在個別控制器的 **Communication Settings → Encrypted Communication** 啟用該裝置加密。

## 4. 常見問題 FAQ

### 4.1 用戶端不支援使用者管理

- **WebServer &lt; V3.5.14.0** 尚不支援使用者管理。
- **對策**：若要讓這些用戶端能連線，控制器上**不要啟用** Enforced user management（改為 Optional）。

### 4.2 用戶端不支援加密通訊

以下用戶端尚不支援加密通訊或有限制：

- Data server（編譯器版本 ≤ V3.5.9.0）
- **WebVisu** &lt; V3.5.14.0 或啟用 file transfer 時
- WebServer &lt; V3.5.14.0
- Remote TargetVisu
- Data source ApplicationV3
- OPC Server V3
- PLCHandler &lt; V3.5.14.0

**對策**：若需讓上述用戶端連線，可將控制器的加密設為 **Optional**，使用戶端可選擇加密或未加密連線。  
**建議**：加密與未加密連線**不要使用同一組帳號密碼**，以降低風險。

### 4.3 專案密碼遺失

- 專案若以密碼加密，密碼遺失後**無法還原**；僅能依賴備份或改用 Dongle／憑證保護的新專案。

### 4.4 憑證有效期

- 首次連線時「安裝為信任」的控制器憑證有效期限較短（例如 30 天）；長期使用請透過 CODESYS Security Agent 或 PLC Shell 設定較長效期或 CA 簽署憑證。

---

## 5. 快速對照

| 目標 | 操作／設定位置 |
|------|----------------|
| 專案加密（密碼／Dongle／憑證） | **Project → Project Settings → Security** |
| 與控制器通訊原則（加密／使用者管理） | 裝置編輯器 **Communication Settings** 或 **Device → Change Communication Policy** |
| 單一控制器啟用加密連線 | 裝置編輯器 **Communication Settings** → **Device** 清單 → **Encrypted Communication** |
| 強制所有連線加密 | **Security Screen → User** → **Enforce encrypted communication** |
| 裝置使用者與群組 | 裝置編輯器 **Users and Groups** 分頁（需先登入裝置使用者） |
| 本機控制器憑證 | Windows `certmgr.msc` → **Controller Certificates** |
