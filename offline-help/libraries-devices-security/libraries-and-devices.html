Libraries and Devices 函式庫與裝置管理 教學整理

本文件基於 CODESYS Offline Help 中下列主題整理而成：

  • Using Libraries（_cds_struct_installing_libraries.html）
  • Managing Devices（_cds_struct_managing_devices.html）
  • Using Library POUs、Adding a Library to the Application/Repository、Library Repository、Updating Library Placeholders
  • Installing Devices、Device Repository 目標是以「團隊共用函式庫與裝置描述」為主線，說明 Library 與 Device 的安裝、Repository 管理與實務注意事項。

1. 整體概念總覽 Overview

在 CODESYS 中，函式庫（Libraries） 與 裝置描述（Devices） 分別由兩套機制管理：

  • Library Repository 函式庫儲存庫：開發系統上存放已安裝函式庫及其版本資訊的位置。專案透過 Library Manager 從儲存庫連結函式庫，才能使用其中的 POUs、資料型別、全域變數等。
  • Device Repository 裝置儲存庫：開發系統上存放已安裝裝置描述檔的位置。專案在 Device Tree 中新增的裝置，必須先在此儲存庫中完成安裝，才能被選用。

實務上，團隊若要共用同一套函式庫或裝置描述，通常會：

  1. 將函式庫／裝置描述安裝到共用的 Repository 路徑（或每台開發機指向同一路徑）。
  2. 注意版本與相容性，必要時使用 Placeholder 與 有效版本（effective version） 管理。

2. Library Repository 函式庫儲存庫

2.1 角色與位置

  • Library Repository 是開發系統上「函式庫檔案與其中繼資料」的存放位置。
  • 透過選單 Tools → Library Repository 可開啟 Library Repository 對話框。
  • 除預設的 System 儲存庫外，可新增、編輯其他儲存庫路徑，供團隊共用或專案專用。
  • 新儲存庫請使用空目錄，或指向既有的有效儲存庫；System 儲存庫不可編輯（以斜體顯示）。

2.2 安裝函式庫到儲存庫

  1. 開啟 Tools → Library Repository。
  2. 在 Location 選擇要安裝到的儲存庫。
  3. 點擊 Install，選擇函式庫檔案（可篩選：All CODESYS libraries、Compiled libraries、Source libraries、All files）。
  4. 點擊 Open 完成安裝；該函式庫會出現在 Installed Libraries 清單，之後即可在 Library Manager 中加入專案。

2.3 從儲存庫加入函式庫到專案（Application）

  1. 在專案中選取 Library Manager（可位於 POUs 視圖或 Devices 視圖下），以 Project → Edit Object 開啟。
  2. 選單 Library → Add Library，在對話框中搜尋並選擇欲使用的函式庫（例如 Util）。
  3. 確認後點擊 OK，函式庫即加入該 Library Manager。

Library Manager 的放置位置會影響可見範圍與 Placeholder 解析順序：

  • 放在 POUs 視圖：該 Library Manager 對所有應用程式可見；Placeholder 僅依 Device 描述或 Library Repository 中的設定解析。
  • 放在 Devices 視圖下：僅該 Device 底下的應用程式可呼叫其函式庫 POUs；Placeholder 會先套用該處的專用解析，再才套用 Device 或 Repository 的設定。

3. Library Manager 與 Placeholder

3.1 Library Manager 功能

  • 顯示專案已整合的函式庫（依類型與屬性）。
  • 可新增、移除函式庫，並編輯函式庫屬性。
  • 若專案以固定版本整合某函式庫，會對應一個 Placeholder library；下載到 PLC 時會載入的版本稱為 effective version 有效版本。
  • Placeholder 的解析可來自：Library Manager 的 Placeholders 對話框、Device 描述、或 Library Repository 中的設定。Library Manager 會顯示實際的 Placeholder 解析與有效版本。

3.2 更新 Placeholder（無法解析時）

若專案使用 Placeholder 函式庫，之後開啟或載入時若無法自動解析，會出現錯誤並在訊息視窗列出受影響的函式庫與應用程式。可採取的作法：

  • Update placeholder <library name> to the newest version：將該 Placeholder 更新為最新版本。
  • Open "Placeholders" dialog of Library Manager：在 Placeholders 對話框中手動設定解析。

建議：若需載入編譯器版本 < 3.5.18.0 的專案，可安裝對應版本的 CODESYS 並用該版本開啟專案，以利 Placeholder 與相容性管理。

4. 在程式中使用函式庫 POU

函式庫可包含：POUs（Function Block、Function）、介面與方法／屬性、資料型別（列舉、結構、別名、聯集）、全域變數／常數／參數列表、文字列表、影像池、視覺化與視覺元素、外部檔案、凸輪表等。

範例：將 Standard 函式庫的計數器 CTUD 加入宣告

  1. 開啟 POU 編輯器，游標放在宣告區。
  2. 輸入實例名稱與冒號（例如 iCounter1:）。
  3. 按 F2 開啟 Input Assistant，在 Structured Types 中選擇 Standard 函式庫下的 CTUD（Counter）。
  4. 勾選 Insert with namespace prefix，按 OK。
  5. 宣告區會出現：iCounter1: Standard.CTUD。

5. Device Repository 裝置儲存庫

5.1 角色與位置

  • CODESYS 以 Device Repository 管理本機已安裝的裝置；預設為安裝時建立的 System 儲存庫（絕對路徑）。
  • 在 Tools → Device Repository 對話框中可安裝／解除安裝裝置；裝置的可組態性、可程式性與可連結的裝置，由 Device Description 裝置描述檔 定義。
  • 安裝後，即可在專案的 Device Tree 中加入該裝置。

注意：請勿手動變更內部裝置儲存庫（勿直接複製檔案進出）。一律透過 Device Repository 對話框安裝或解除安裝裝置。

5.2 安裝裝置描述

  1. Tools → Device Repository。
  2. 選擇安裝位置（預設為 System Repository）。
  3. 點擊 Install，在 Install Device Description 對話框中選擇裝置描述檔路徑與檔案類型篩選（例如 *.devdesc.xml，或廠商格式如 PROFIBUS DP 的 *.gsd、CAN 的 *.eds／*.dcf）。
  4. 選擇要安裝的裝置描述後點擊 Open。
  5. CODESYS 會將裝置描述加入對應類別；若安裝過程有錯誤（例如參照的檔案遺失），會在對話框下方顯示。

安裝時 CODESYS 會將裝置描述檔與參照檔案複製到內部位置，因此事後修改原始檔不會影響已安裝的裝置；若有修改，需重新安裝，並建議更新裝置描述中的內部版本號。

5.3 裝置儲存庫其他操作

  • Location / Edit：顯示與編輯儲存庫路徑；可 Add／Edit／Remove 儲存庫位置（Remove 可選擇只從清單移除或一併刪除磁碟上的目錄）。
  • Installed Device Descriptions：以樹狀列出已安裝裝置（Name、Vendor、Version）；可依字串篩選、依 Vendor 篩選。
  • Uninstall：移除所選裝置；移除後該裝置無法再在程式系統中使用。
  • Renew Device Repository：更新儲存庫內所有裝置（例如匯入外掛更新後，過期描述會以符號標示，可由此更新）。
  • Download Missing Descriptions：專案使用到儲存庫中沒有的裝置時會出現；可從清單選擇並下載缺少的裝置描述。

6. 版本與相容性注意事項

項目建議
函式庫版本團隊共用時建議固定函式庫版本或使用 Placeholder＋明確解析策略，避免不同成員編譯出不同 effective version。
裝置描述修改裝置描述後請更新內部版本號並重新安裝，以利辨識與更新。
Repository 路徑共用路徑時確保所有開發機皆有讀寫權限，且勿手動改動 Library/Device Repository 內部檔案。
編譯器版本舊專案（編譯器 < 3.5.18.0）建議以對應版本的 CODESYS 開啟，以正確處理 Library Profile 與 Placeholder。

7. 快速操作對照

目的操作入口
安裝函式庫到本機Tools → Library Repository → Install
將函式庫加入專案開啟 Library Manager → Library → Add Library
管理 PlaceholderLibrary Manager → Placeholders 對話框；或訊息視窗中的 quickfix
安裝裝置描述Tools → Device Repository → 選擇 Location → Install
在專案使用裝置Device Tree 右鍵 → Add Device，從已安裝的裝置中選擇
更新裝置儲存庫Device Repository → Renew Device Repository

上述流程可支援「集中安裝函式庫／裝置 → 多專案或多成員共用」的維運情境；若需批次或自動化，可再搭配 CODESYS Script 或 Command-line 使用（詳見 Packages、Licenses 與 Scripts 教學文件）。
