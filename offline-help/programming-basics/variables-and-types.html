Variables and Types 變數與型別 教學整理

本文件基於 CODESYS Offline Help 中下列主題整理而成：

  • Variables（_cds_struct_reference_variable_types.html）
  • 各變數類型：VAR、VAR_INPUT、VAR_OUTPUT、VAR_IN_OUT、VAR_GLOBAL、VAR_TEMP、VAR_STAT、VAR_EXTERNAL、VAR_INST、VAR_CONFIG、CONSTANT、VAR_GENERIC CONSTANT
  • PERSISTENT、RETAIN 目標是從「變數生命週期與儲存區」角度整理各關鍵字的作用與適用情境，並以表格與簡短 ST 範例說明；詳細語法請參閱 Offline Help 各變數類型頁面。

1. 變數範圍（Scope）與宣告

變數的範圍（scope）決定「可在哪裡、以何種方式」使用該變數，並在變數宣告中定義。宣告區依關鍵字區分為多種區塊（VAR、VAR_INPUT、VAR_GLOBAL 等），每種區塊對應不同的生命週期、儲存位置與可見範圍。

2. 變數類型與儲存區一覽

關鍵字適用位置生命週期／儲存典型用途
VARPOU 內該 POU 每次被呼叫時存在；呼叫結束後不保證保留區域變數、POU 內部暫存
VAR_INPUTFB / PROGRAM 等由呼叫端傳入，唯讀（除非 VAR_IN_OUT）輸入參數
VAR_OUTPUTFB / PROGRAM 等由 POU 寫入，回傳給呼叫端輸出參數
VAR_IN_OUTFB / PROGRAM 等傳參考，讀寫皆影響呼叫端變數雙向參數（in-out）
VAR_GLOBAL全域變數列表（GVL）整個應用程式執行期間存在，所有 POU 可存取跨 POU 共用資料
VAR_TEMPPOU 內僅在單一週期／單一呼叫內有效，不保證跨週期保留暫存計算結果、迴圈計數等
VAR_STATPOU 內（如 FB）在呼叫之間保持值，同一實例內持久FB 的靜態區域變數（狀態保持）
VAR_EXTERNALPOU 內引用外部（如 GVL）已宣告的全域變數使用 GVL 變數而不重複宣告
VAR_INSTFB 內FB 實例的成員變數，與 VAR 類似但語意為「實例變數」FB 內部狀態、子實例
VAR_CONFIG專案／裝置層組態時設定，Runtime 可讀裝置參數、可組態常數
CONSTANTPOU 或 GVL編譯期常數，不可寫入常數
VAR_GENERIC CONSTANT泛型 FB／Library泛型參數，編譯時決定型別或值泛型程式設計

3. PERSISTENT 與 RETAIN

兩者皆與「斷電或重啟後是否保留值」有關，但語意與實作不同：

修飾詞保留時機典型儲存使用情境
PERSISTENT值寫入非揮發性儲存，PLC 斷電或冷啟動後仍可保留（依裝置與設定）非揮發性儲存區需跨斷電保留的計數、設定、歷史資料
RETAIN在**暖啟動（Warm Restart）**或部分斷電情境下保留；冷啟動行為依 Runtime／裝置而定保留區（retentive 區）重啟後需還原的狀態、不需永久寫入非揮發性儲存時

實務建議：

  • 若裝置明確區分「retentive 記憶體」與「persistent／非揮發性儲存」，請依裝置文件與 CODESYS 專案設定確認冷／暖啟動後的行為。
  • 重要資料若需確保持久，應確認 PERSISTENT 的寫入時機與備份策略（若有）；RETAIN 通常用於「同一程式重載或暖啟動後還原」的狀態。

4. 生命週期概念簡述

  • VAR：POU 區域變數，每次呼叫可重新初始化（依實作）；若為 FB 實例內的 VAR，其生命週期跟隨該 FB 實例。
  • VAR_TEMP：僅在單一掃描週期或單次 POU 執行內有效，不跨週期保留。
  • VAR_STAT：在 FB 或具狀態的 POU 中，於呼叫與呼叫之間保持值，同一實例內相當於「靜態區域變數」。
  • VAR_GLOBAL：程式啟動時存在，直到程式結束；所有 POU 皆可存取（需注意多任務同時寫入的同步問題）。

5. 簡短 ST 範例

區域變數與輸入輸出（FB 內）：

FUNCTION_BLOCK FB_Example
VAR_INPUT
    inEnable : BOOL;
    inSetpoint : INT;
END_VAR
VAR_OUTPUT
    outActual : INT;
    outDone : BOOL;
END_VAR
VAR
    internalCount : INT;   (* 區域變數 *)
END_VAR
VAR_STAT
    lastValue : INT;       (* 呼叫間保持 *)
END_VAR

全域變數與 RETAIN（GVL 內）：

VAR_GLOBAL
    g_TotalCount : DINT;
    RETAIN g_RetainCounter : INT;   (* 暖啟動後保留 *)
END_VAR

常數與 PERSISTENT（概念）：

CONSTANT
    MAX_VALUE : INT := 1000;
END_CONSTANT

VAR PERSISTENT
    p_RecipeId : DINT;     (* 斷電後可保留，依裝置與設定 *)
END_VAR

需求建議關鍵字
POU 內部暫存、不跨呼叫保留VAR 或 VAR_TEMP（依是否跨週期）
FB 狀態在呼叫間保持VAR_STAT
跨 POU 共用VAR_GLOBAL（在 GVL 宣告）；或 VAR_EXTERNAL 引用
暖啟動後保留RETAIN
