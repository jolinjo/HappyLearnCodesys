CNC10_DynamicPath — 整體架構與功能說明

一、這個專案在做什麼？

本範例示範如何在 不使用 G-code 檔案或 CNC 物件的情況下，由 PLC 程式在執行期間動態產生 CNC 路徑。Path 程式使用陣列 xp / yp 定義一串目標點，依序組裝 SMC_GeoInfo 並追加到 SMC_OUTQUEUE 佇列，經 SMC_CheckVelocities 檢查後提供給 Ipo 程式的 SMC_Interpolator 使用；Ipo 再透過二軸龍門座標轉換與位置控制驅動 X_Drive / Y_Drive。

三、控制流程：由淺入深

3.1 Path：動態組裝路徑佇列

Path.st 的主要內容（含前面我們已加的中文註解）可簡化描述如下：

PROGRAM Path
VAR
	iState: INT;
	CheckVel: SMC_CheckVelocities;

	QUEUE: SMC_OUTQUEUE;
	BUF: ARRAY[0..99] OF SMC_GEOINFO;
	GEO: SMC_GEOINFO := (... 預設速度、加減速 ...);
	n: INT := 0;
	xp: ARRAY[1..7] OF REAL := [100,50,50,10,90,0,0];
	yp: ARRAY[1..7] OF REAL := [0,100,0,50,100,100,0];
END_VAR

流程：

  1. 狀態 0：初始化佇列
    • QUEUE.pbyBuffer := ADR(BUF[0]);
    • SM3_CNC.SMC_SetQueueCapacity(ADR(QUEUE), SIZEOF(BUF))，指定佇列容量。
    • iState := 1。
  2. 狀態 1：填滿佇列
    • 在 WHILE NOT QUEUE.bFull 迴圈中：
      • n := n + 1;，依序處理 xp[n] / yp[n]。
      • GEO.iSourceLine_No := n;。
      • GEO.piStartPos := GEO.piDestPos;，上一點為本段起點。
      • GEO.iMoveType := LIN;，設定線性移動。
      • GEO.iObjNo := GEO.iObjNo + 1;。
      • GEO.piDestPos.dX := xp[n]; GEO.piDestPos.dY := yp[n];。
      • SMC_CalcLengthGeo(ADR(GEO))，計算幾何長度。
      • SMC_AppendObj(ADR(QUEUE), ADR(GEO))，追加到佇列。
      • 如果 n 已達陣列長度，設定 QUEUE.bEndOfList := TRUE; iState := 2; EXIT;。
  3. 狀態 2：完成
    • 不再新增幾何段，維持佇列內容。
  4. 速度檢查
    • 每個週期呼叫 CheckVel(bExecute := TRUE, poqDataIn := ADR(QUEUE));，輸出路徑供插補器使用。

總結來說，Path 將一組點列（xp/yp）動態轉成 CNC 路徑，而非依賴固定的 G-code 檔案。

3.2 Ipo：插補、龍門座標轉換與軸控制

Ipo.st：

PROGRAM Ipo
VAR
	trafo: SMC_TRAFO_Gantry2;
	p1,p2: SMC_ControlAxisByPos;
	trafof: SMC_TRAFOF_Gantry2;
	smci: SMC_Interpolator;
	mcp1, mcp2: MC_Power;
END_VAR

典型實作流程（與其他 CNC 範例類似）：

  1. 兩軸上電：
    • mcp1(Enable := TRUE, bRegulatorOn := TRUE, bDriveStart := TRUE, Axis := X_Drive);
    • mcp2(Enable := TRUE, bRegulatorOn := TRUE, bDriveStart := TRUE, Axis := Y_Drive);
  2. 插補器：
    • smci 以 poqDataIn := Path.CheckVel.poqDataOut 為輸入（實際程式在 Interpolation() 中呼叫），將 OutQueue 轉成連續的 SetPosition（X/Y）。
  3. 座標轉換：
    • trafo / trafof 將插補輸出的機械座標轉成 X/Y 驅動軸的 SetPosition，支援龍門結構與視覺化。
  4. 軸控制：
    • p1 / p2（SMC_ControlAxisByPos）以 fSetPosition 寫入 X_Drive / Y_Drive。

四、各支程式負責哪些功能？

4.1 Ipo.st

  • 類型：PROGRAM。
  • 職責：
    • 上電 X/Y 兩軸。
    • 從 Path 產生的路徑佇列插補 SetPosition。
    • 經二軸龍門座標轉換後驅動兩軸。

4.2 Path.st

  • 類型：PROGRAM。
  • 職責：
    • 動態組裝 SMC_OUTQUEUE：以 xp / yp 陣列產生線段幾何段（SMC_GeoInfo），設定速度、加減速與目標座標。
    • 透過 SMC_SetQueueCapacity、SMC_AppendObj 控制佇列容量與內容。
    • 使用 SMC_CheckVelocities 對整條路徑作速度檢查，輸出給插補器。

七、特別的演算法與觀念

7.1 為什麼要動態產生路徑？

相較於從 G-code 檔案讀取，本範例的做法適用於：

  • 路徑由 PLC 計算（例如依感測器輸入、配方或線上演算法）。
  • 外部系統直接傳入點列（table of points），PLC 端再組裝成幾何段。
  • 需要在運轉中隨時插入、修改、重建路徑的應用。

這種「程式組裝幾何段」的方式比編輯 G-code 更易於與控制邏輯整合。

7.2 OutQueue 的結構與生命週期

  • SMC_OUTQUEUE 代表一條待插補的路徑序列，可以重複被 CheckVel / Interpolator 消費。
  • bEndOfList := TRUE 標記路徑已經到結尾。
  • 可以在執行期間重建 QUEUE，例如改變 xp/yp 後重新進入狀態 0/1 產生新路徑。

八、重要參數與設定位置

參數檔案 / 位置說明
xp / yp 內容Path.st定義整條路徑的形狀。
GEO.dVel/dAccel/dDecelPath.st 中 GEO 預設值每段線段的速度與加減速設定。
OutQueue 容量Path.st 中 BUF 大小與 SMC_SetQueueCapacity決定最多可緩衝多少幾何段。

九、建議閱讀與修改順序

  1. 先讀 Path.st，理解狀態機與 QUEUE / GEO / xp / yp 的關係。
  2. 再讀 Ipo.st，與 CNC01 的 Ipo 對照，確認插補與二軸控制結構。
  3. 在 CODESYS 中將 xp/yp 改為其他點，觀察路徑與運動結果變化。
  4. 若要加入圓弧或樣條，可在產生 GEO 時改用對應的 iMoveType 與參數。
