PLCopenSingle2 — 整體架構與功能說明

一、這個專案在做什麼？（一句話＋情境）

這個範例示範如何在同一個程式中，事先準備三個常用的運動控制功能塊：MC_Power、MC_MoveVelocity、MC_MoveAbsolute，並透過初始化參數設定好「上電」、「速度模式運動」與「絕對定位」的預設行為。

你可以把它想成「先把所有運動指令的模板設定好」，再由其他程式或可視化畫面依需求去啟動對應的命令。

二、使用情境與硬體／通訊架構

2.1 硬體與軸配置

DeviceTree_Axes.md 顯示本範例只有一根虛擬軸：

軸名稱類型綁定任務
DriveSM_Drive_Virtual-

同樣是使用虛擬軸 Drive 來模擬實際驅動器行為，方便在沒有硬體的情況下做程式測試。

2.2 任務與週期

從 TaskConfiguration.md 可以看到有兩個任務：

任務名稱類型週期優先權POU 清單
TaskCyclict#4ms1MOTION_PRG
VISU_TASKCyclict#100ms10VisuElems.Visu_Prg
  • Task 每 4ms 呼叫一次 MOTION_PRG，主要負責運動相關的運算與狀態更新。
  • VISU_TASK 每 100ms 呼叫一次 VisuElems.Visu_Prg，通常用來更新可視化畫面，例如顯示軸位置或按鈕狀態。

2.3 資料流與角色分工

  • 任務 Task 調用 MOTION_PRG，裡面準備了三個運動功能塊的執例。
  • 可視化程式 VisuElems.Visu_Prg 可能透過變數或參照，去操作這些功能塊的 Execute 或設定值（實際細節依專案原始設計而定）。
  • 軸 Drive 負責執行這些命令並回報實際位置／速度等資訊。

三、控制流程：由淺入深

3.1 MOTION_PRG 的角色

開啟 MOTION_PRG.st 可以看到，程式中目前只包含變數宣告區塊，沒有進一步的流程控制或狀態機。這代表它在這個範例裡扮演的是「集中宣告與初始化運動控制功能塊」的角色：

  • mcp：負責上電與啟動軸。
  • mcmv：負責速度模式運動。
  • mcma：負責絕對位置運動。

實際的呼叫順序、觸發時機，很可能由其他程式（特別是可視化）來決定，MOTION_PRG 只負責「把功能塊準備好、設定好預設值」。

3.2 速度模式與絕對定位的概念

這個範例同時準備了速度模式與絕對定位兩種指令：

  • 速度模式（MC_MoveVelocity）：

    • 給定一個速度與加減速度後，軸會持續以該速度旋轉或移動，適合用在「連續輸送帶」或「需要連續運轉」的場景。

  • 絕對定位（MC_MoveAbsolute）：

    • 給定一個目標位置，軸會以指定的速度與加減速度移動到該點，適合用在「點到點定位」或「需要精準位置」的場景。

初學者可以把這兩種功能塊的設定值當成「運動菜單」的預設參數，之後只要改變 Execute 或目標位置，就能反覆使用。

四、各支程式負責哪些功能？（由淺入深）

本目錄的程式檔案主要是 MOTION_PRG.st，其餘可視化程式在專案中以名稱 VisuElems.Visu_Prg 存在，但不在此目錄。

4.1 MOTION_PRG.st

  • 類型：PROGRAM
  • 職責：
    • 集中宣告並初始化三個運動控制功能塊：MC_Power、MC_MoveVelocity、MC_MoveAbsolute。
    • 為每一個功能塊預先設定合理的速度、加速度、減速度與目標位置等參數。
  • 與硬體／通訊關係：
    • 功能塊實際連結的軸為 Drive，在 Device 樹中對應到 SM_Drive_Virtual。
    • 其他程式（如視覺化程式）可以透過共享變數或全域參照來操作這些功能塊。

五、函式庫與函式／功能塊一覽（用途、引數、回傳）

5.1 函式庫清單（對照 Libraries.md）

庫名稱解析版本／備註
#SM3_Basic（系統庫，提供基本運動控制功能塊，如 MC_Power、MC_MoveVelocity、MC_MoveAbsolute）
#SM3_CamBuilder（系統庫，本範例未直接使用其凸輪功能）
#System_VisuElem3DPath（系統庫，提供 3D 路徑視覺化元件）
#System_VisuElemCamDisplayer（系統庫，提供顯示 Cam 曲線的視覺化元件）

5.2 功能塊與主要引數

5.2.1 MC_Power()

  • 用途、引數與輸出說明可參考 PLCopenSingle 範例，本處同理。
  • 在此範例中，mcp 預設 Enable := TRUE，讓上電請求在專案啟動後立即成立。

5.2.2 MC_MoveVelocity()

  • 用途：
    • 讓軸以指定速度連續運動，直到外部條件（例如 Execute := FALSE）結束動作。
  • 主要引數（僅列出本範例有設定者）：
    • Velocity : LREAL — 目標速度，程式中設為 100。
    • Acceleration / Deceleration : LREAL — 速度變化的加速度與減速度。
    • Direction : MC_Direction — 運動方向，本範例設為 positive。
  • 回傳型態：VOID（功能塊）。

5.2.3 MC_MoveAbsolute()

  • 與 PLCopenSingle 範例類似，但此處預設位置與速度略有不同：
    • Position 設為 180。
    • 速度與加減速度同樣為 100，方向為 positive。

六、變數與常數一覽

本範例未使用獨立 GVL 檔案，所有變數皆在 MOTION_PRG.st 中宣告，亦未使用 RETAIN / PERSISTENT。

6.1 MOTION_PRG.st 內的變數

名稱類別型態預設值說明
mcp區域變數MC_PowerEnable := TRUE，其他為預設軸上電與啟動用的功能塊執例。
mcmv區域變數MC_MoveVelocityVelocity := 100，Acceleration/Deceleration := 100，Direction := positive速度模式運動的功能塊執例。
mcma區域變數MC_MoveAbsolutePosition := 180，Velocity/Acceleration/Deceleration := 100，Direction := positive絕對定位運動的功能塊執例。

七、特別的演算法與觀念

7.1 以「初始化」為主的程式寫法

雖然 MOTION_PRG 目前沒有流程控制邏輯，但它示範了一種常見的寫法：
先在程式裡集中宣告所有會用到的運動控制功能塊，並透過結構化的初始化語法，直接給定每個功能塊的預設參數。

這樣做有幾個優點：

  • 不需要在程式的多個地方重複設定相同的速度／加速度。
  • 所有關鍵運動參數都在同一個地方，方便日後調整。
  • 其他程式只要控制 Execute 或個別引數，就可以重複使用這些預設設定。

八、重要參數與設定位置（實作時對照）

8.1 軸與任務

項目名稱說明
軸Drive在 Device 樹中設定為 SM_Drive_Virtual，對應到本範例中的單一虛擬軸。
任務Task週期 4ms，負責呼叫 MOTION_PRG。
任務VISU_TASK週期 100ms，負責呼叫 VisuElems.Visu_Prg（視覺化）。

8.2 運動參數

  • 速度模式（mcmv）：Velocity := 100，Acceleration/Deceleration := 100。
  • 絕對定位（mcma）：Position := 180，Velocity/Acceleration/Deceleration := 100。

實務上若想要改變運動特性，可以直接在 MOTION_PRG.st 的初始化區塊中調整這些數值。

九、建議閱讀與修改順序（給初學者）

  1. 先看 MOTION_PRG.st 的變數區塊
    • 理解 mcp、mcmv、mcma 各自對應的功能塊與初始化參數。
  2. 再對照第五章的功能塊說明
    • 把每個初始化欄位（例如 Velocity、Position）與實際運動行為連結起來。
  3. 最後在 CODESYS 中開啟 VISU 畫面（若有）
    • 觀察按鈕或控制項如何操作這些功能塊，強化對「程式參數」與「畫面操作」之間關係的理解。
