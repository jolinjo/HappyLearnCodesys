# Project Export, Transfer, Compare and Protection 專案匯出、移轉、比較與保護 教學整理

> 本文件依據 Offline Help 中 `Exporting and Transferring Projects`、`Exporting and Importing Projects`、`Transferring Projects`、`Comparing Projects` 以及部份專案保護相關章節整理而成，說明專案在團隊合作與維護情境下的實務流程。

---

## 1. 專案分享與移轉概觀 Overview

在實務開發中，常見的情境包括：

- 將專案交給 **另一位工程師或團隊** 接手維護。
- 在 **新電腦** 上繼續對相同 PLC 做維護，盡量避免 Online Change / 重新下載。
- 將部分程式或 POU 以 **XML 形式** 匯出，與其他工具或專案共享。
- 比較兩個版本的專案差異，作為程式碼審查或版本回溯依據。

對應到 CODESYS 的功能：

- 專案檔案與封存：
  - `*.project`、`*.projectarchive`
- 物件層級匯出 / 匯入：
  - `*.export`（CODESYS XML）
  - `*.xml`（PLCopenXML）
- Project Archive（專案封存）：
  - 包含裝置描述、Library profile、下載資訊、Visualization Profile 等。
- Project Comparison（專案比較）：
  - 比較兩個專案或專案版本的差異。

---

## 2. 專案匯出與匯入 Exporting and Importing Projects

### 2.1 使用 Project 檔案與 Archive 檔案

根據 `Exporting and Transferring Projects`：

- 在兩套 CODESYS 開發系統之間交換專案時，最常見方式是複製：
  - 專案檔：`*.project`
  - 專案封存：`*.projectarchive`
- 一般建議：
  - 若只在同一環境內備份，可使用 `*.project`。
  - 若需跨電腦／跨團隊交接，建議使用 **Project Archive**，因為可額外打包：
    - 下載資訊（download information files）。
    - 參考的裝置（Referenced devices）。
    - 參考的函式庫（Referenced libraries）。
    - Visualization Profile 等。

### 2.2 物件級匯出：CODESYS XML 與 PLCopenXML

`Exporting and Importing Projects` 章節指出：

- **CODESYS XML (*.export)**：
  - 完全相容於 CODESYS 專案格式。
  - 物件以機器可解析的 XML 型式儲存。
  - 適合在 **CODESYS 專案之間** 交換程式碼或物件。
- **PLCopenXML (*.xml)**：
  - 方便與其他程式編輯器或文件工具交換資訊。
  - 但只支援 PLCopen 定義的一部分元素，與 CODESYS 完整功能不一定百分之百相容。

#### 匯出物件步驟（Export）

1. 開啟專案。
2. 在主選單選擇：
   - `Project → Export`（CODESYS XML），或
   - `Project → Export PLCopenXML`。
3. 在 `Export` / `Export PLCopenXML` 對話框中選擇要匯出的物件。
4. 按 `OK`，並在檔案對話框選擇檔名與儲存位置。

#### 匯入物件步驟（Import）

1. 開啟欲匯入目標的專案。
2. 在主選單選擇：
   - `Project → Import`，或
   - `Project → Import PLCopenXML`。
3. 在匯入對話框中：
   - 選擇來源檔案與欲匯入之物件。
   - 指定匯入目標位置（在物件樹中的節點）。
4. 按 `OK`，匯入的物件會新增到現有物件樹內。

實務建議：

- 在團隊共用 Library / 範本程式時：
  - 優先使用 `*.export`（CODESYS XML），保留最多資訊。
  - 只有在與第三方工具整合時，才採用 PLCopenXML。

---

## 3. 專案移轉與維護情境 Transferring Projects

`Transferring Projects` 章節特別針對以下需求設計：

> 將專案從 PC1 轉移到 PC2，並在 PC2 上連到 **同一台 PLC**，而且希望「不需要 Online Change 或重新 Download」。

### 3.1 準備條件（在 PC1 上）

在 PC1 開啟專案並連到目標 PLC，依序確認：

1. **函式庫版本固定（Fix Version Libraries）**
   - 在 `Library Manager` 中確認所有 Library 項目都不是 `*`（代表自由 placeholder），而是固定版號。
   - 例外：純介面 Library 可以維持為彈性版本。
2. **Compiler Version 固定**
   - 在 `Project → Project Settings → Compiler Options` 中設定為特定版本。
3. **Visualization Profile 固定（若有使用 Visualization）**
   - 在 `Project Settings → Visualization Profile` 中選擇明確的 Profile。
4. **Boot Application 與專案內容一致**
   - 檢查視窗標題列中的專案名稱後是否有 `*`：
     - 有 `*` 代表尚未儲存，可能與已下載的 Boot Application 不一致。
   - 必要時：
     1. 先儲存專案。
     2. `Online → Create Boot Application` 建立新的 Boot Project。
     3. `Online → Login` 後執行 `Online → Download` 重新下載到 PLC。
     4. `Debug → Start` 讓 PLC 上執行的應用程式與專案完全一致。

### 3.2 建立 Project Archive

在上述條件都滿足後：

1. 執行 `File → Project Archive → Save/Send Archive`。
2. 在 `Project Archive` 對話框中勾選：
   - `Download information files`
   - `Referenced devices`
   - `Referenced libraries`
   - `Visualization Profile`
   - 若使用較舊 Compiler，可能仍看到 `Library profile` 選項（在 3.5.18.0 之後已不再適用）。
3. 將 Archive 檔案儲存到 PC2 可存取的位置（例如網路路徑或版本控制系統）。

### 3.3 在 PC2 上還原與登入

1. 在 PC2 開啟 CODESYS。
2. `File → Project Archive → Extract Archive`：
   - 選擇 PC1 儲存的 Archive 檔。
   - 在 `Extract Project Archive` 對話框中，勾選與建立封存時相同的項目。
3. 開啟還原後的專案。
4. 設定與原先相同的通信設定（IP / Device），登入同一台 PLC。

若環境（Library / Compiler / Visualization Profile 等）完全相容：

- CODESYS 應該可以在 PC2 上直接登入，而 **不需要 Online Change 或 Download**。

## 4. 專案比較 Comparing Projects

`Comparing Projects` 章節提供一個非常實用的工具：**Project Comparison**。

### 4.1 功能概觀

- 可以將目前開啟的專案與另一個「參考專案（Reference Project）」比對。
- 會偵測並顯示：
  - 內容差異（程式碼、設定）。
  - 物件屬性差異。
  - 存取權限差異。
- 比對結果會顯示在：
  - `Project Comparison – Differences` 主視圖（樹狀結構）。
  - 針對單一物件的 `Project Comparison – <object name> Differences` 詳細視圖（程式碼逐行比較）。

### 4.2 啟動比較

1. 開啟當前專案（作為「目前專案」）。
2. 選擇 `Project → Compare`。
3. 在 `Project Comparison` 對話框中：
   - 指定要比較的「參考專案」（另一個 `.project` 或 `.projectarchive` 解開的專案）。
   - 設定欲比對的範圍與選項（例如是否包含圖形編輯器、權限設定等）。
4. 執行比對後，`Differences` 視圖會以樹狀結構標示：
   - 新增的物件。
   - 刪除的物件。
   - 內容不同的物件（以顏色與圖示區分）。

### 4.3 詳細差異檢視與接受變更

對於內容不同的物件，可以開啟詳細視圖：

- 顯示「目前專案」與「參考專案」的程式碼差異。
- 可以使用工具列上的圖示（例如「接受此區塊」、「接受單行」、「接受屬性差異」）將參考專案的內容帶入目前專案：
  - 被標記為接受的差異會以黃色顯示。
- 一次只能在 **一個比較視圖** 中編輯與接受差異，其餘比較視圖會是唯讀狀態。

當你關閉（或離開）該比較視圖並確認要接受所選差異時：

- CODESYS 會將那些變更 **實際套用到目前專案**。

實務用法範例：

- **Code Review / Merge**：
  - 把同事修改後的專案當作「參考專案」，用 Project Comparison 比對差異，逐項接受或拒絕。
- **版本回溯檢查**：
  - 將新舊版本專案比較，確認哪些邏輯或設定被改動。

---

## 5. 專案保護與國際化的連結（概念預告）

雖然完整的專案保護、在地化與範本（Template）會在其他教學文件中詳述，這裡先點出與「匯出／比較／移轉」的關聯：

- **保護與儲存 Protecting and Saving Projects**
  - 當你建立 Project Archive 或分享專案時，應檢查：
    - 專案是否設有唯讀保護或密碼。
    - 是否啟用 User Management 限制部分物件的存取。
  - 在交接前可以適度放寬或重新規劃權限，以便接手者有效維護。
- **在地化 Localizing Projects**
  - 多語系專案常常需要在不同地區團隊間傳遞。
  - 建議在 Archive 中一併納入 Text List / String Table 等資源，並於教學文件中清楚說明翻譯流程。
- **專案範本 Project Templates**
  - 當你找到一套穩定的專案結構（含 Library、Device、User Management、檔案組織方式），可以將之固化為 Template。
  - 未來新專案不必從零開始，也能保持團隊風格一致。

---

## 6. 實務流程建議：專案交接 Checklist

當你準備把專案交給另一個團隊或搬到新電腦時，可以參考以下流程：

1. **在 PC1 檢查專案狀態**
   - Library 皆為 Fix Version。
   - Compiler / Visualization Profile 已固定。
   - Boot Application 與專案內容一致並已下載到目標 PLC。
2. **建立 Project Archive**
   - 使用 `Save/Send Archive`，勾選 Download Info / Referenced Devices / Referenced Libraries / Visualization Profile 等選項。
   - 將 Archive 上傳至版本控制系統或安全儲存位置。
3. **記錄環境資訊**
   - CODESYS 安裝版本與 Add‑ons 版本（可由 CODESYS Installer 匯出描述檔）。
   - 目標 PLC 的 Runtime 版本與 Device Description 版本。
4. **在 PC2 還原並驗證**
   - Extract Archive。
   - 依既有記錄建立或選擇相容的 CODESYS 安裝（可透過 Installer 協助）。
   - 連線同一台 PLC，確認可以登入且不需要 Online Change / Download。
5. **使用 Project Comparison（可選）**
   - 若在 PC2 進行修改，可在交回給原團隊前，以 Project Comparison 比對差異並整理變更紀錄。

透過這樣的流程，可以在 CODESYS 環境中穩定地分享、移轉與比較專案，並與後續的專案保護與國際化策略銜接，形成一套完整的專案生命週期管理方式。
