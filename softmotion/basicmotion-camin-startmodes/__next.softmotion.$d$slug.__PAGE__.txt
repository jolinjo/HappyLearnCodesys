# BasicMotion_CamIn_StartModes — 整體架構與功能說明

---

## 一、這個專案在做什麼？

這個範例示範在同一條主軸（<span class="cfg-name">Master</span>）上，如何用三種不同的 <span class="cfg-func">MC_CamIn()</span> 啟動模式來啟動三個從軸，並透過實際 Cam 運動的差異，幫助你理解「凸輪啟動條件、加減速方式與啟動距離」之間的關係。  
主軸以速度模式持續運轉，三個從軸分別搭配不同的 CamTable 與 CamIn 設定，讓你可以在視覺化與 Trace 中直接比較各種 Start Mode 的行為差異。

---

## 二、使用情境與硬體／通訊架構

實務上，CamIn 常用在需要「週期性重複且可預先定義軌跡」的應用，例如：包裝機上的貼標輪、間歇送料、旋轉切刀等。  
本範例以全虛擬軸模擬這種情境，只專注在 CamIn 的設定與行為觀察，不牽涉真實 I/O。

### 2.1 軸與任務配置

根據 `DeviceTree_Axes.md` 與 `TaskConfiguration.md`，本範例軸與任務如下：

| 軸名稱 | 類型 | 角色 | 綁定任務 |
|--------|------|------|----------|
| <span class="cfg-name">Master</span> | <span class="cfg-name">SM_Drive_Virtual</span> | 主軸，提供凸輪運動的時間與位置基準 | （由 SoftMotion 內部更新） |
| <span class="cfg-name">SlaveAbsolute</span> | <span class="cfg-name">SM_Drive_Virtual</span> | 從軸 1，以「絕對模式」啟動 CamIn | 同上 |
| <span class="cfg-name">SlaveRampIn</span> | <span class="cfg-name">SM_Drive_Virtual</span> | 從軸 2，以「RampIn」方式平順進入 Cam 運動 | 同上 |
| <span class="cfg-name">SlaveRampInDist</span> | <span class="cfg-name">SM_Drive_Virtual</span> | 從軸 3，以「RampInDist」設定指定距離內完成進入凸輪 | 同上 |

任務設定如下（來自 `TaskConfiguration.md`）：

| 任務名稱 | 類型 | 週期 | 優先權 | POU 清單 |
|----------|------|------|--------|----------|
| MainTask | Cyclic | t#20ms | 1 | `PLC_PRG` |

MainTask 每 20 ms 週期性呼叫 `PLC_PRG`，在此任務中完成：

- 主軸上電與速度控制。  
- 讀取主軸實際位置。  
- 三組從軸的 CamTableSelect 與 CamIn 控制。  

### 2.2 函式庫

根據 `Libraries.md`，本範例使用：

| 庫名稱 | 解析版本／備註 |
|--------|----------------|
| <span class="cfg-name">#SM3_Basic</span> | SoftMotion 基本運動控制功能塊（MC\_*）。 |
| <span class="cfg-name">#SM3_CamBuilder</span> | 提供 CamTable 建立與管理的相關功能。 |

---

## 三、控制流程：由淺入深

### 3.1 整體流程概觀

可以用下面這句話記住整體流程：

> 啟動主軸 → 持續讀取主軸位置 → 根據觸發條件選擇 CamTable 並啟動三組 CamIn → 比較不同 Start Mode 的跟隨行為。

概略步驟如下（實際細節在 `PLC_PRG` 中實作）：

1. **主軸上電與啟動**：透過 <span class="cfg-func">MC_Power()</span> 讓 <span class="cfg-name">Master</span> 軸進入可受控狀態，並使用 <span class="cfg-func">MC_MoveVelocity()</span> 讓主軸以固定速度轉動。  
2. **讀取主軸位置**：利用 <span class="cfg-func">MC_ReadActualPosition()</span> 持續更新主軸實際位置，作為後續 CamIn 判斷的依據。  
3. **選擇 CamTable**：對三個從軸，各自呼叫 <span class="cfg-func">MC_CamTableSelect()</span>，指定要使用哪一組凸輪表（例如一個完整週期的從軸位移曲線）。  
4. **等待啟動條件**：通常會有一個「開始凸輪」的條件，例如外部命令或主軸位置到達某範圍；本範例使用 <span class="cfg-func">R_TRIG()</span> 偵測啟動信號的上升沿。  
5. **三種 CamIn 模式比較**：  
   - <span class="cfg-name">SlaveAbsolute</span>：在特定主軸位置立即以「絕對模式」進入 Cam 運動。  
   - <span class="cfg-name">SlaveRampIn</span>：在一定時間／距離內平順「拉入」凸輪軌跡。  
   - <span class="cfg-name">SlaveRampInDist</span>：指定在主軸的某一段距離內完成由自由運動到凸輪運動的過渡。  
6. **觀察與比較**：  
   - 在視覺化與 Trace 中，同時觀察三個從軸的位置與速度，感受 Start Mode 差異。  
   - 調整 CamIn 的參數（啟動位置、Ramp 斜率、距離等），體驗對運動品質的影響。

### 3.2 R\_TRIG 的角色

在 `PLC_PRG` 中，<span class="cfg-func">R_TRIG()</span> 物件 <span class="cfg-local">R_TRIG_StartCamming</span> 負責將「開始啟動 Cam」的信號轉成單次脈衝：  

- 當外部條件從 FALSE 變成 TRUE 時，<span class="cfg-func">R_TRIG()</span> 的輸出只會在那一個週期變成 TRUE。  
- 這樣一來，即使啟動條件維持 TRUE 多個週期，CamIn 也只會被觸發一次，避免重複進入或重置。  

對初學者來說，可以把它想像成「按一下按鈕只算一次，長按不會重複觸發」。

## 四、各支程式負責哪些功能？

本範例只有一個程式 `PLC_PRG`，但是內部整合了多個功能塊；理解時可以把它拆成幾個區塊來看。

### 4.1 `PLC_PRG` — 主控制與 CamIn 示範

- **類型**：PROGRAM。  
- **職責**：  
  - 管理 Master 與三個從軸的上電與速度控制。  
  - 讀取主軸實際位置，作為 CamIn 啟動判斷依據。  
  - 對三個從軸分別選擇凸輪表並設定 CamIn 參數。  
  - 以三種不同 Start Mode 啟動 CamIn，讓使用者比較從軸運動差異。  
- **與硬體／通訊的關係**：  
  - 在實務架構中，可對應到 EtherCAT 伺服驅動器上的主軸與三個從軸；本範例使用 <span class="cfg-name">SM_Drive_Virtual</span> 模擬。  

閱讀建議：  
- 先看變數宣告區，了解每個軸對應到哪個功能塊實例。  
- 再往下看「上電與速度控制」、「讀取主軸位置」、「CamTableSelect 與 CamIn 設定」等區塊，對照程式註解與本文件的第五、六章內容。

## 七、特別的演算法與觀念

### 7.1 CamIn Start Mode 的觀念

雖然程式主要是設定與呼叫功能塊，但背後有幾個重要觀念：

- **主軸作為「時間軸」**：在 Cam 應用中，主軸位置通常就代表「時間推進」，所有從軸都跟著主軸的位置走。  
- **Start Position／Start Distance**：決定從軸何時開始從「自由運動」轉為「依照凸輪表運動」。  
- **RampIn / RampInDist**：決定這個轉換是立刻完成，還是用某個斜率或距離「拉入」到凸輪軌跡上，避免太劇烈的速度／加速度變化。  

在本範例中，你可以把三個從軸想像成「同一條生產線上三種不同品牌的機構」，它們做的工作一樣，但因為啟動方式不同，跟隨主軸的感覺也不一樣。

### 7.2 利用 R\_TRIG 避免重複啟動

<span class="cfg-func">R_TRIG_StartCamming()</span> 的目的，是把「條件為 TRUE 的一段時間」轉換成「啟動瞬間的一個脈衝」，這樣做有兩個好處：

1. CamIn 只會在你「打算啟動」的那一刻執行一次，不會在條件維持 TRUE 期間被重複觸發。  
2. 在 Trace 中，更容易看出「啟動瞬間」與之後的凸輪運動。  

這種寫法在很多運動控制專案中都很常見，是很值得熟悉的一個小技巧。

## 八、重要參數與設定位置

雖然本範例沒有額外的 GVL 常數，但有幾個地方很值得你在實驗時多試著調整：

| 參數／設定 | 所在位置 | 作用說明 |
|------------|----------|----------|
| 主軸速度（<span class="cfg-arg">Velocity</span>） | `PLC_PRG` 中的 <span class="cfg-local">mvMaster</span> 初始值 | 決定主軸轉動速度，會直接影響從軸跟隨的快慢。 |
| 主軸加減速度／Jerk | `PLC_PRG` 中的 <span class="cfg-local">mvMaster</span> 初始值 | 影響主軸運動的平順程度。 |
| 各 CamIn 的 Start Position／Start Distance | `PLC_PRG` 中對三個 <span class="cfg-local">ciSlave*</span> 的設定 | 決定從軸何時開始進入凸輪運動。 |
| CamTable 選擇 | `PLC_PRG` 中三個 <span class="cfg-local">ctsSlave*</span> 的設定 | 決定從軸在一個主軸週期內的位移曲線形狀。 |
| MainTask 週期 t#20ms | `TaskConfiguration.md` | 影響功能塊更新頻率與數值解析度。 |

在實務機台上，通常會先固定主軸速度與任務週期，再透過調整 CamTable 與 CamIn 的 Start Mode 來達到理想的運動品質。

## 九、建議閱讀與修改順序

1. **先看 `DeviceTree_Axes.md` 與 `TaskConfiguration.md`**  
   - 了解有哪些軸、它們扮演什麼角色、任務週期與優先權為何。  
2. **再看 `PLC_PRG` 的變數宣告區**  
   - 把每一個功能塊實例與實際軸一一對應起來，先不用急著理解所有參數，只要知道「誰負責哪個軸」。  
3. **接著看主軸的上電與運轉區塊**  
   - 對照本文件第五章中 <span class="cfg-func">MC_Power()</span> 與 <span class="cfg-func">MC_MoveVelocity()</span> 的說明，理解啟動流程。  
4. **然後看主軸位置讀取與 R\_TRIG 區塊**  
   - 理解為什麼要用 <span class="cfg-func">MC_ReadActualPosition()</span> 與 <span class="cfg-func">R_TRIG()</span>，再對照 Trace 觀察位置與啟動瞬間。  
5. **最後看三組 CamTableSelect 與 CamIn 設定**  
   - 將三個從軸的設定放在一起比較：哪些參數一樣、哪些不同。  
   - 嘗試只改動其中一個從軸的參數，再下載執行，觀察運動差異。  

這樣的閱讀順序可以讓你先建立「主軸時間軸」的概念，再一步一步加上凸輪與從軸的行為，比直接看完整程式容易許多。
