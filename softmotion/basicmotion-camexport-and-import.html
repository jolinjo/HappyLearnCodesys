BasicMotion_CamExportAndImport — 整體架構與功能說明

一、這個專案在做什麼？

這個範例示範如何在 CODESYS SoftMotion 中，使用 CamBuilder 以程式方式建立一條凸輪曲線，並透過 SMC_WriteCAM() 匯出成檔案，再使用 SMC_ReadCAM() 從檔案匯入回系統。
換句話說，它教你「如何把記憶體中的 Cam 轉成檔案」以及「如何把檔案中的 Cam 再讀回來」，方便在實機專案中管理、備份或交換凸輪表。

二、使用情境與硬體／通訊架構

在實務應用中，越複雜的機械越需要根據產線需求調整凸輪曲線（例如改變加速度、動作角度、同步範圍等）。將凸輪存成檔案可以帶來幾個好處：

  • 方便在不同機台之間交換同一組 Cam 設定。
  • 可以在現場調整後匯出備份，未來需要時再匯入。
  • 可由上位系統或離線工具產生 Cam 檔，再由 PLC 匯入使用。

本範例不直接操作軸，只專注在「Cam 資料結構 ↔ 檔案」的轉換，因此 DeviceTree_Axes.md 中沒有實際解析到軸；你可以把它視為「純資料處理、與軸脫鉤」的示範。

三、控制流程：由淺入深

3.1 一句話版流程

進入程式後，自動建立一條簡單的凸輪 → 檢查是否有錯誤 → 若成功則匯出成 CAM 檔 → 再從該檔案把凸輪讀回來，確認檔案可正確使用。

3.2 狀態機步驟說明

PLC_PRG 使用 state 作為狀態機，重要狀態如下（對照原始程式）：

  1. STATE_CREATE_CAM（建立凸輪）

    • 呼叫 camBuilder.Init() 清空建構器。
    • 依序 Append 三個 Segments：
      • 第 1 段：Poly5()，起／終點皆採用 BoundImplicit()，表示與前一段／預設條件平順銜接。
      • 第 2 段：Line()，從 (160,140) 線性移動到 (200,220)，並指定加速度邊界。
      • 第 3 段：Poly5()，最後收斂到 (360,360) 並讓末端速度為 0。
    • 檢查 camBuilder.IsErrorPending() 是否有錯誤；若無，呼叫 SMCB.InitCamRef() 初始化 camRefExport，再呼叫 camBuilder.Write(camRefExport) 將凸輪資料寫入緩衝區。
    • 若寫入成功（error = SMC_NO_ERROR），狀態轉為 STATE_EXPORT_CAM，否則進入錯誤路徑。

  2. STATE_EXPORT_CAM（匯出 CAM 檔案）

    • 呼叫 SMC_WriteCAM()（變數名 writeCam），指定 cam=camRefExport、bExecute=TRUE、sFileName='exportedCam.cam'。
    • 若 writeCam.bDone 為 TRUE，表示匯出完成，狀態轉為 STATE_START_IMPORT_CAM。
    • 若 writeCam.bError 為 TRUE，表示匯出失敗，狀態轉為錯誤路徑。

  3. STATE_START_IMPORT_CAM（從檔案匯入凸輪）

    • 呼叫 SMC_ReadCAM()（變數名 readCam），指定 bExecute=TRUE、sFileName='exportedCam.cam'。
    • 若 readCam.bDone 為 TRUE，狀態轉為 STATE_DONE，代表完成整個建立＋匯出＋匯入流程。
    • 若 readCam.bError 為 TRUE，則進入錯誤狀態。

  4. STATE_DONE / STATE_ERROR

    • STATE_DONE：流程結束，通常保持在此狀態，讓使用者在監看視窗中確認結果。
    • STATE_ERROR：說明在建立、匯出或匯入任一階段發生錯誤，可進一步顯示 error 或 writeCam／readCam 的錯誤碼來除錯。

整體來說，這個狀態機是「單趟流程」：程式啟動一次後跑完建立與讀寫檔案，之後就停在 STATE_DONE 或 STATE_ERROR，方便你在監看畫面確認結果。

四、各支程式負責哪些功能？

本範例只有一個程式 PLC_PRG，但裡面整合了幾個核心元件。

4.1 PLC_PRG — 凸輪建立與匯出／匯入示範

  • 類型：PROGRAM。
  • 主要職責：
    • 使用 CamBuilder 建立一條簡單但具有代表性的凸輪曲線。
    • 將凸輪內容寫入 MC_CAM_REF 所指向的緩衝區。
    • 呼叫 SMC_WriteCAM() 匯出為 CAM 檔案。
    • 呼叫 SMC_ReadCAM() 從檔案匯入，驗證檔案可以被完整讀回。
  • 適合學習的觀點：
    • 從這個程式你可以學到「如何組裝 Cam 資料結構」以及「如何控制寫檔與讀檔流程」，而不被軸或機構細節干擾。

5.2 主要函式與功能塊

名稱類型用途主要引數（IN / IN_OUT）回傳／狀態
SMCB.CamBuilder類別／物件以程式方式建立一條凸輪，內部維護 Segments 清單。Init()：重設內容；Append()：加入一段 Segment；Write()：將內容寫入 CamRef。IsErrorPending()：檢查是否有錯誤；Write() 回傳 SMC_ERROR。
SMCB.Poly5()函數建立五次多項式型態的 Segment，適合做平順的起動／停止。BoundStart、BoundEnd：起終點邊界條件。傳回一個 SMC_CAM_SEGMENT。
SMCB.Line()函數建立線性 Segment，在主軸座標與從軸座標間做直線插補。BoundStart、BoundEnd。傳回一個 SMC_CAM_SEGMENT。
SMCB.Bound()函數建立具體的邊界條件，如 (MasterPos, SlavePos, AccLimit, JerkLimit)。位置與限制條件。傳回一個邊界結構。
SMCB.BoundImplicit()函數使用隱含邊界條件（例如「延續前一段的速度」），常用於段與段之間平順銜接。無顯式位置輸入。傳回一個邊界結構。
SMCB.InitCamRef()函數將 MC_CAM_REF 指向一個 Segments 陣列與其大小。CamRef、pSegments、sizeBytes。將 CamRef 結構填妥，無明確回傳值。
SMC_WriteCAM()FUNCTION_BLOCK將 MC_CAM_REF 內容寫成 CAM 檔案。cam：來源凸輪；bExecute：觸發寫檔；sFileName：檔名。bDone：寫檔完成；bError：寫檔錯誤。
SMC_ReadCAM()FUNCTION_BLOCK從 CAM 檔讀回凸輪並載入系統。bExecute；sFileName。bDone：讀檔完成；bError：讀檔錯誤。

七、特別的演算法與觀念

7.1 使用 CamBuilder 組裝凸輪

這個範例的精華在於：「不用手動編輯 CAM 檔，而是在程式裡組裝出一條凸輪曲線」。
關鍵步驟如下：

  1. 呼叫 camBuilder.Init() 清空建構器。
  2. 使用多個 Append()，把 Poly5() 與 Line() 建立的 Segments 依序加入。
  3. 確認 IsErrorPending() 沒有錯誤。
  4. 呼叫 SMCB.InitCamRef() 設定好 camRefExport 指向 camSegmentsExport。
  5. 呼叫 Write() 把內容填到 camRefExport，準備給 SMC_WriteCAM() 使用。

初學者可以這樣理解：
「CamBuilder 就像是『凸輪編輯器』的後端 API，你在程式中一段段描述主從軸的關係，它幫你整理成可以匯出或被軸使用的資料格式。」

7.2 記憶體結構與檔案之間的關係

MC_CAM_REF 扮演的角色，是把「程式記憶體中的 Segments 陣列」包裝成一個標準介面，供讀寫函式塊（例如 SMC_WriteCAM()）使用。
透過這個設計，你可以：

  • 將 CamBuilder 建好的凸輪直接餵給軸的 Cam 運動功能塊。
  • 或是先匯出成檔案、再在另一台機器上匯入。

這個範例專注在第二種情境：確認「從記憶體 → 檔案 → 記憶體」的往返流程是可行的。

八、重要參數與設定位置

雖然本範例不驅動實際軸，但還是有幾個關鍵參數會影響凸輪形狀與檔案管理：

參數／設定所在位置作用說明
MAX_NUM_OF_CAM_SEGMENTSPLC_PRG 常數區陣列大小，決定最多可以容納幾段 Segments。
EXPORT_CAM_FILE_NAMEPLC_PRG 常數區匯出與匯入使用的檔名，實務上可改成有意義的名字（例如依機種或工單命名）。
SMCB.Line() 與 SMCB.Poly5() 的邊界條件PLC_PRG 中的 STATE_CREATE_CAM決定凸輪曲線在主軸座標與從軸座標上的關係，以及平順程度。
MainTask 週期 t#20msTaskConfiguration.md影響建構與匯出／匯入的觸發頻率（本例中主要是流程邏輯，對數值精度影響不大）。

在實際專案中，你可以把這個範例中的常數與邊界條件改成「真實機械的凸輪參數」，再匯出給現場工程師或上位系統使用。

九、建議閱讀與修改順序

  1. 先讀 PLC_PRG 中的變數與常數宣告
    • 搞清楚 camBuilder、camSegmentsExport、camRefExport、writeCam、readCam 各自扮演的角色。
  2. 再看 STATE_CREATE_CAM 區塊
    • 對照本說明第七章，理解 Poly5() 與 Line() 這三段 Segments 的差異，並試著修改邊界參數觀察效果。
  3. 接著看 STATE_EXPORT_CAM 與 STATE_START_IMPORT_CAM
    • 理解 SMC_WriteCAM() 與 SMC_ReadCAM() 的呼叫方式與錯誤處理。
  4. 最後試著修改檔名與錯誤處理行為
    • 例如改成在錯誤狀態中重試、或是把錯誤碼輸出到某個監看變數中。

這樣的順序可以讓你先掌握資料流與物件關係，再動手改動重要參數而不會迷路。
